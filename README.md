## 代码逻辑

整个程序的运行流程可以概括为 **"初始化 → 交互 → 计算 → 渲染"** 四步循环：

```
程序启动
  │
  ├── 1. ti.init(arch=ti.gpu)          # 初始化 GPU 后端（CUDA/Vulkan/Metal）
  │
  ├── 2. init_particles()              # [GPU] 并行初始化所有粒子的位置和速度
  │
  └── 3. 主循环（每帧重复）──────────────────────────────────
        │                                                   │
        ├── 3a. gui.get_cursor_pos()   # [CPU] 获取鼠标坐标  │
        │                                                   │
        ├── 3b. update_particles()     # [GPU] 物理计算:     │
        │       ├─ 计算每个粒子到鼠标的距离和引力              │
        │       ├─ 更新速度（引力加速 + 阻力衰减）            │
        │       ├─ 更新位置（pos += vel × dt）               │
        │       └─ 边界碰撞检测与反弹                         │
        │                                                   │
        ├── 3c. pos.to_numpy()         # GPU→CPU 数据回传    │
        │                                                   │
        └── 3d. gui.circles()          # [CPU] 绘制粒子      │
              └── gui.show()           # 刷新窗口 ───────────┘
```

## 实现功能

### 1. GPU 并行粒子模拟
将 10000 个粒子的物理计算编译为 GPU 着色器，所有粒子**同时并行**更新，而非逐个串行计算。在支持 CUDA 的显卡上，帧率可达 60FPS+。

### 2. 实时鼠标交互
粒子受鼠标位置产生的引力场影响，鼠标移动时粒子群会实时追随，形成类似万有引力的群集效果。鼠标静止时粒子会在其周围形成聚集，移动时粒子群拖出流体般的尾迹。

### 3. 物理真实感
通过引力加速、空气阻力衰减和弹性碰撞三个机制的组合，粒子运动呈现出自然的动力学行为：加速、减速、反弹，而非简单的匀速直线运动。

## 操作说明

| 操作 | 效果 |
|------|------|
| 移动鼠标 | 粒子群受引力吸引，追随鼠标运动 |
| 快速移动鼠标 | 粒子群被甩出，形成流体拖尾效果 |
| 鼠标静止 | 粒子逐渐聚集在鼠标周围 |
| 关闭窗口 | 退出程序 |
